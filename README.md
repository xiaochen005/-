# 视频分享平台 - Streamlit 网页项目
## 项目简介
基于 Python + Streamlit + SQLite 开发的轻量化在线视频分享平台，支持用户注册登录、视频上传/播放、分类展示、评论弹幕、收藏、回收站软删除、管理员后台、操作日志、OSS云端存储全套功能。
无需复杂部署，一键上传至 Streamlit Cloud 即可在线访问，数据库自动初始化，开箱即用。

## 核心功能清单
### 用户端
1. 账号系统：新用户注册、密码MD5加密存储、登录状态会话保存
2. 首页展示：已上线视频卡片、播放量排序、自动过滤回收站视频
3. 视频操作：本地视频+封面上传、OSS云端存储、在线播放
4. 互动功能：视频评论、实时弹幕、收藏/取消收藏
5. 分类筛选：视频分类、二级分类筛选浏览
6. 个人中心：查看我的上传、我的收藏、修改信息

### 管理员端（默认账号：admin，密码：123456）
1. 视频管理：审核视频、下架视频、移入回收站、永久删除
2. 回收站：统一管理标记删除视频，支持恢复/彻底清除
3. 用户管理：查看所有注册用户
4. 系统日志：记录登录、注册、上传、删除等全部操作记录
5. 数据统计：视频总量、用户数量、播放量概览

### 数据库特性
1. 自动建库建表，启动自动补齐缺失字段（`is_delete`/`sub_category`）
2. 软删除机制：视频仅标记删除，不物理销毁，可恢复
3. 完整数据表：用户、视频、评论、弹幕、收藏、操作日志、举报、合集分组
4. 线程安全数据库连接，适配 Streamlit 多会话运行

## 本地运行教程
1. 克隆/下载项目到本地
2. 安装依赖
```bash
pip install -r requirements.txt
```
3. 启动项目
```bash
streamlit run main.py
```
4. 访问地址：http://127.0.0.1:8501
5. 管理员账号：`admin` 密码：`123456`

## Streamlit Cloud 云端部署教程
1. 将项目上传至 GitHub 仓库（**不要上传 database/video_web.db**）
2. 登录 Streamlit Cloud，新建 App，关联该 GitHub 仓库
3. 高级配置 Secrets，填入阿里云 OSS 密钥（oss_upload.py 所需）
4. 启动文件填写 `main.py`，点击部署
5. 部署完成后自动生成在线访问链接

### 部署注意事项
1. `.gitignore` 配置屏蔽数据库文件，云端每次重启自动全新生成数据库
2. 项目启动自动执行 `init_db`，自动创建完整数据表、补齐缺失字段
3. 密码统一MD5加密存储，线上线下登录逻辑完全一致
4. 日志表字段统一修正，登录、操作不会触发SQL字段报错
5. 解决循环导入问题，启动无 ImportError 崩溃

## 数据库设计说明
1. user 用户表：用户名、加密密码、管理员权限、注册时间
2. video 视频表：标题、简介、分类、封面地址、OSS视频地址、播放量、上传人、审核状态、软删除标记
3. comment 评论表：绑定视频ID、评论人、内容、时间
4. danmaku 弹幕表：视频ID、发送用户、弹幕内容、播放时间点、颜色
5. collect 收藏表：用户-视频收藏关联
6. operation_log 操作日志表：操作类型、操作用户、描述、操作时间
7. report、collection_group、group_video_rel 配套扩展表

## 关键技术点
1. MD5 密码加密，明文不入库，保障账号安全
2. SQLite 自动兼容旧表，自动 ALTER 新增缺失字段
3. Streamlit Session State 会话管理，保持登录状态
4. 阿里云OSS对象存储，解决云端本地文件无法持久保存问题
5. 软删除设计，回收站可恢复数据，避免误删丢失内容
6. 统一数据库工具封装，分页查询、日志写入、增删改查统一接口
7. 完全修复循环导入、SQL字段不匹配、字段缺失、账号密码不匹配等线上报错

## 常见问题解决方案
1. 管理员账号密码错误
   原因：初始化存入明文，登录使用MD5校验 → 已修复，init_db 内置加密函数，管理员密码加密入库
2. 启动 ImportError 循环导入
   原因：db_utils 与 init_db 互相引用 → init_db 内置独立加密函数，解除双向导入
3. 登录时报 sqlite3.OperationalError
   原因：日志表字段名不匹配 → write_log SQL 修正字段 opt_user
4. 首页加载数据库报错找不到 is_delete 字段
   原因：云端旧表缺少字段 → init_db 启动自动检测并新增字段
5. 页面无限加载/部署卡死
   禁止上传 database/video_web.db 到仓库，云端自动生成全新数据库
